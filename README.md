# Claude Code × Python おまかせ開発 — 題材リポジトリ

Udemy コースで使用する題材フォルダ一式です。題材は `materials/` フォルダに入っています。お好きな方法で入手してください。

## 入手方法

### 方法 1: Download ZIP（git を使わない方）

1. このページ上部の緑色の **「Code」ボタン** をクリック
2. **「Download ZIP」** を選択
3. ダウンロードした ZIP を解凍し、フォルダをデスクトップなど分かりやすい場所に置く
4. 解凍したフォルダの中の `materials/` を開くと、題材フォルダが並んでいます

### 方法 2: git clone（git に慣れている方）

```
git clone https://github.com/zunda-lab/claude-code-python-materials.git
```

クローン後、題材は `claude-code-python-materials/materials/` にあります。

## 題材一覧（`materials/` 内）

| フォルダ | 内容 |
|----------|------|
| `01_hello_world` | はじめての題材（Hello World） |
| `02_odd_even` | 偶数・奇数の判定 |
| `03_weekday` | 曜日の判定 |
| `04_excel_worktime` | Excel 勤務時間の集計 |

## 使い方

各題材フォルダ**そのもの**を VSCode で開き、「コンテナーで再度開く（Reopen in Container）」を実行してください。
詳しい手順はコース内のレクチャーで解説しています。

## 初回起動時の注意（macOS）

題材フォルダをデスクトップなどに置いて「コンテナーで再度開く（Reopen in Container）」すると、
**初回だけ** macOS が次のような許可ダイアログを出すことがあります。

> 「"Docker" が "デスクトップ" フォルダ内のファイルにアクセスしようとしています」

これは Docker が題材フォルダを読むために必要なので、**「許可」**を選んでください（2回目以降は出ません）。

もし許可する前にビルドが進んでしまい、
**「マウント構成が無効です」/「bind source path does not exist」** といったエラーで失敗した場合は、
許可したうえで通知の **「再試行」** を押せば、そのまま続行できます。

> デスクトップ・書類・ダウンロードは macOS が保護するフォルダのため、この確認が出ます。
> 気になる場合は、題材をホーム直下の任意のフォルダ（例: `~/dev`）に置くと確認が出ません。
